# 해빗트래커 앱 요구사항 정의서

## 프로젝트 개요

매일의 습관을 기록하고 체크하는 모바일 앱. 다람쥐가 도토리를 모으는 게임 요소를 통해 습관 달성의 재미를 더한다.

---

## 기술 스택

| 역할 | 기술 |
|------|------|
| 프레임워크 | Expo (React Native) + TypeScript |
| 라우팅 | Expo Router |
| UI 컴포넌트 | Gluestack UI v2 |
| 스타일링 | NativeWind (Tailwind CSS) |
| 서버 상태 | TanStack Query (React Query) |
| 클라이언트 상태 | Zustand |
| 백엔드 / DB | Supabase |
| 애니메이션 | Reanimated 3 |
| 아이콘 | Lucide Icons |

---

## 화면 목록

| 화면 | 경로 | 설명 |
|------|------|------|
| 로그인 / 회원가입 | `/auth` | Supabase Auth 기반 인증 |
| 홈 (오늘의 습관) | `/` | 오늘 해야 할 습관 목록 및 체크 |
| 달력 뷰 | `/calendar` | 날짜 선택 후 해당 날 습관 조회 및 체크 |
| 습관 관리 | `/habits` | 습관 추가 / 수정 / 삭제 |
| 월간 통계 | `/stats` | 달성률, 스트릭, 도토리 통계 |

## 기능 요구사항

### AUTH — 인증

| ID | 기능 | 설명 |
|----|------|------|
| AUTH-001 | 회원가입 | 이메일 + 비밀번호로 신규 계정 생성 |
| AUTH-002 | 로그인 | 이메일 + 비밀번호로 로그인 |
| AUTH-003 | 로그아웃 | 세션 종료 |
| AUTH-004 | 자동 로그인 | 앱 재실행 시 로그인 상태 유지 |

---

### HABIT — 습관 관리

| ID | 기능 | 설명 |
|----|------|------|
| HABIT-001 | 습관 추가 | 이름, 반복 요일(월~일 복수 선택) 입력 후 저장 |
| HABIT-002 | 습관 수정 | 이름 및 반복 요일 변경 |
| HABIT-003 | 습관 삭제 | 습관 및 관련 체크 기록 삭제 |

---

### CHECK — 일별 체크

| ID | 기능 | 설명 |
|----|------|------|
| CHECK-001 | 오늘의 습관 조회 | 오늘 요일에 해당하는 습관 목록 표시 |
| CHECK-002 | 습관 체크 / 해제 | 완료 여부 토글, Supabase에 즉시 저장 |
| CHECK-003 | 날짜 선택 조회 | 달력에서 특정 날짜 선택 후 해당 날 습관 조회 |
| CHECK-004 | 과거 날짜 수정 | 과거 날짜의 체크 기록 수정 가능 |

---

### STAT — 월간 통계

| ID | 기능 | 설명 |
|----|------|------|
| STAT-001 | 습관별 달성률 | 해당 월 기준 `완료일 / 해야 할 일수 × 100` (%) |
| STAT-002 | 전체 평균 달성률 | 모든 습관의 달성률 평균 |
| STAT-003 | 최장 연속 스트릭 | 습관별 가장 긴 연속 달성 일수 |
| STAT-004 | 최고 / 최저 달성 습관 | 이달 가장 잘 지킨 습관과 가장 못 지킨 습관 하이라이트 |
| STAT-005 | 과거 월 조회 | 월 선택하여 지난 달 통계 조회 |

---

### GAME — 게임화 (도토리 시스템)

| ID | 기능 | 설명 |
|----|------|------|
| GAME-001 | 도토리 적립 | 습관 1개 체크 완료 시 도토리 1개 적립 |
| GAME-002 | 보너스 도토리 | 하루 목표 습관 전체 달성 시 보너스 도토리 3개 적립 |
| GAME-003 | 황금 도토리 | 7일 연속 전체 달성 시 황금 도토리 1개 적립 |
| GAME-004 | 획득 애니메이션 | 도토리 획득 시 통통 튀는 이펙트 애니메이션 표시 |
| GAME-005 | 월간 도토리 통계 | 월간 통계 화면에 이달 총 도토리 수 표시 |

## 데이터 모델

### users (Supabase Auth 기본 제공)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | 사용자 고유 ID |
| email | text | 이메일 |

### habits
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | 습관 고유 ID |
| user_id | uuid | 사용자 ID (FK) |
| name | text | 습관 이름 |
| target_days | int[] | 반복 요일 (0=일 ~ 6=토) |
| created_at | timestamp | 생성일 |

### habit_logs
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | 로그 고유 ID |
| habit_id | uuid | 습관 ID (FK) |
| user_id | uuid | 사용자 ID (FK) |
| date | date | 체크 날짜 (YYYY-MM-DD) |
| completed | boolean | 완료 여부 |

### acorns (도토리)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid | 고유 ID |
| user_id | uuid | 사용자 ID (FK) |
| type | text | `normal` / `golden` |
| reason | text | 적립 사유 (GAME-001 ~ 003) |
| earned_at | timestamp | 획득 일시 |

## 비기능 요구사항

| 항목 | 내용 |
|------|------|
| 플랫폼 | iOS / Android |
| 오프라인 | 네트워크 없을 시 안내 메시지 표시 |
| 보안 | Supabase RLS(Row Level Security)로 타 유저 데이터 접근 차단 |
| 성능 | 체크 반응 속도 300ms 이내 |
